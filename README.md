# Crime Data Analysis in India (2001–2012)

Tools: Tableau, KNIME  
Focus: Crimes against women  
Output: Interactive dashboards and analytical insights
Dataset: 12 years of crime statistics across Indian states (published by the National Crime Records Bureau (NCRB), Government of India)

This project demonstrates a complete analytics workflow from
data preprocessing to insight generation using visual dashboards.

![Dashboard Preview](https://github.com/SanaNarmawala/crime-analysis/blob/main/dashboards/dashboard_crime_trends.jpg)


## Project Overview

This project explores crime trends in India over a 12-year period using a structured data analytics workflow involving **data cleaning, parameter reduction, preprocessing simulation, and interactive dashboard visualization**.

The analysis focuses on identifying **patterns in crimes against women across different states and years**, and presenting insights through visual analytics dashboards.

The goal of the project was to demonstrate how data-driven analysis can uncover **regional disparities, long-term crime trends, and evolving patterns in reported incidents**.

---

## Objectives

• Analyze long-term crime trends across India  
• Identify states with the highest reported incidents  
• Understand how different crime categories evolve over time  
• Build interactive dashboards to visualize patterns in crime data  
• Extract insights that could support data-driven policy and awareness initiatives

---

## Dataset

The dataset contains **12 years of reported crime statistics across multiple states in India**.

Key attributes in the dataset include:

- State / Region  
- Year  
- Crime Category  
- Number of Reported Incidents  

The analysis particularly focuses on **crimes against women**, examining how their occurrence varies across geography and time.

## Data Analysis Pipeline

The project follows a structured analytics workflow:

1. **Data Collection**  
   Crime dataset covering multiple states across a 12-year period.

2. **Data Cleaning**  
   Data inconsistencies and formatting issues were cleaned and standardized to ensure accuracy.

3. **Parameter Reduction**  
   Relevant attributes were selected to reduce dataset complexity and focus on meaningful analytical variables.

4. **Preprocessing Simulation (KNIME)**  
   Data cleaning and preprocessing workflows were simulated using KNIME to model an automated data preparation pipeline.

5. **Visualization & Dashboard Development**  
   Interactive dashboards were created in Tableau to analyze crime patterns and trends.

6. **Insight Extraction**  
   Observed patterns in crime distribution across states and years were analyzed to derive insights.

## Key Insights

• Certain states consistently report significantly higher incidents of crimes against women.

• Kidnapping and abduction cases show noticeable growth across multiple years.

• Some crime categories remain relatively stable, while others demonstrate clear upward trends.

• Long-term analysis reveals regional disparities in crime reporting and highlights areas that may require targeted interventions.
